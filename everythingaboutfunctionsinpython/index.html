Defining your first function in Python

Let’s start by defining our first function that accomplishes the simple task of printing “Hello, world” to the screen.

def print_hello_world():
    print("Hello, world")print_hello_world()

Let’s break this down:

  • We start by telling the interpreter that we are defining a function using the def keyword
  • Then comes the name of the function which in our case is print_hello_world . You can define it as you like
  • Then there are two round brackets. Currently, these are empty but they contain any parameters that you want to pass to a function (more on that in the next section)
  • Then there is an indented block of code containing the code to be executed in the function. In our case, it’s just printing “Hello, world”.
  • Till now, we have defined our function. It won’t print anything unless we call it which we do at the end.
  • Similarly, we can call the function 10 times and it will print “Hello, world” 10 times but the definition would just be written once. This is the power of functions in terms of usability

Note that a function generally has a return statement too. This is the value that the function returns and can be stored as you’d like.

Parameters in Functions

Parameters, or arguments, are values that you can pass to a function that will determine how a function will get executed. There are different ways on how we can pass the parameters:

Positional Parameters:

The most common type of passing parameters is by calling a function and passing the parameters in the same position as in the definition of the function. Let’s take an example of a division function:

def integer_division(num_1, num_2):
    return num_1 // num_2

If we want to divide the number 10 by number 2, we need to call the function as follows:

integer_division(10, 2)

This will now return 5. But if we change the position like this:

integer_division(2, 10)

Then the function will return 0, so the position is of importance when passing positional parameters. Note that we need to pass the two required parameters here, otherwise we will get a TypeError indicating that we have passed an incorrect number of parameters

Keyword Parameters:

We can also pass the parameters in a key=value format when calling a function. This means we don’t require to keep the sequence in mind. Consider the same function as above:

def integer_division(num_1, num_2):
    return num_1 // num_2

We can call this using keyword parameter like this:

integer_division(num_1=10, num_2=2)

Or like this:

integer_division(num_2=2, num_1=10)

Both of these will return 5. We can also pass a combination of positional and keyword arguments too. But the condition is that the keyword arguments will appear after all the positional arguments like follows:

integer_division(10, num_2=2)

This is required so that the Python interpreter can make sense of the sequence and assign the correct values to each parameter

Default Arguments:

Python also allows us to define default arguments when defining functions as well. We can define these during the function definition like this:

def integer_division(num_1=12, num_2=2):
    return num_1 // num_2

Now we can call the function as follows:

integer_division()

Which will return 6 but we can also pass custom values too (all are valid choices):

integer_division(10) -> returns 5
integer_division(12, 3) -> returns 4
integer_division(num_2=4) -> returns 3
integer_division(10, num_2=5) -> returns 5

Note: There is a common problem with the default arguments when you pass a mutable object such as a list to a function as a default argument. Consider the following function:

def append_item(a_list=[]):
    a_list.append('item')
    print(a_list)

You can call the function as follows with a list:

append_item(['item_1'])
append_item(['item_2'])

This is as expected. Every time a new list is created and we append an item to it. But let’s see what happens when we use the default argument:

append_item()
append_item()
append_item()

Shocked, right? I was too!!. It should have printed a list with a single item each time.

The reason behind this is that the default argument is created once when the function is defined and is stored in memory. That’s why using a mutable object modifies that object in memory and can result in some unexpected results. A better way for this function would be as follows:

def append_item(a_list=None):
    if a_list is None:
        a_list = []
  
    a_list.append('item')
    print(a_list)

This will ensure that a new empty list is created everytime when using default argument.

Variable Length Positional Arguments (*args):

You may have seen this ( *args ) before but don’t know what that means. This is used when we don’t know the exact number of arguments we would pass but need to process them in a similar manner.

For instance, you want to write a function for adding some numbers. You don’t know in advance how many numbers you are going to add. This comes in handy in cases like this:

def add(*args):
    total = 0
    for num in args:
        total += num
    
    return total

When you use this syntax, the args is basically a tuple of values of all the positional arguments. Let’s call this function:

add(1, 2, 3, 4, 5)

This will return 15 as expected. We can even print the tuple and check for ourselves. Also, the args name is not necessary, you can define it as you’d like but with an asterisk(*).

def add(*list_of_numbers):
    total = 0
    print(list_of_numbers)
    for num in list_of_numbers:
        total += num    return total

Variable Length Keyword Arguments(**kwargs):

We can also pass an arbitrary number of keyword arguments and the function will receive those as a dictionary, from which we can extract the values we want:

def print_kwargs(**kwargs):
    print(kwargs)

We can call this function in multiple ways:

print_kwargs(num_1=10, num_2=5, num_3=1)
print_kwargs(name='Nouman', age='23', profession='Software Engineer')
